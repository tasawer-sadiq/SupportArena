# AI for Civic Innovation Hackathon 2026

## What to do next

1. Understand the hackathon requirements
   - Build a working prototype.
   - Write a clear project description.
   - Include a meaningful AI component.
   - Create a demo video (up to 3 minutes).
   - Prepare a presentation deck.
   - Share the source code repository link.

2. Define your problem statement
   - Choose one of the themes: Democracy & Civic Participation, Technology for Civic Good, or Open Data & Access to Information.
   - Focus on a specific public challenge, not the whole sector.
   - Clearly describe:
     - The problem being solved.
     - Why it matters.
     - Who benefits.

3. Choose the target user group
   - Identify the primary audience (citizens, community groups, NGOs, government staff, etc.).
   - Make the use case simple and realistic.
   - Example: help citizens access local service information, verify civic news, or improve a small community service.

4. Plan the solution and AI use
   - Define the core feature set you can build quickly.
   - Pick an AI function that adds real value, such as:
     - text summarization or translation for civic information,
     - document classification for open-data transparency,
     - chatbot assistance for citizen services,
     - image/video analysis for community reporting.
   - Explain why AI is appropriate and how it improves the solution.

5. Build the prototype first
   - Focus on a working demo over many features.
   - Develop a minimal viable product (MVP) you can show in a demo video.
   - Keep the user flow short and easy to understand.

6. Prepare the second-round submission
   - Devpost registration and submission:
     - Create a Devpost account.
     - Submit your project through the official hackathon page.
   - Include in Devpost:
     - Project title and team name.
     - Problem statement.
     - Solution overview.
     - Key features.
     - Team member details.
     - Screenshots, wireframes, mockups, repository link.
   - Make sure the submission reflects actual progress.

7. Create supporting materials
   - Project description document or README.
   - Demo video showing:
     - the problem,
     - your solution,
     - key features,
     - how users interact with it.
   - Presentation deck with:
     - problem,
     - solution,
     - impact,
     - roadmap.

8. Keep the June 13 deadline in mind
   - Prioritize tasks that can be completed before June 13.
   - Do not spend too much time on features that cannot be finished.
   - Aim for a strong proof of concept and clear presentation.

## Recommended action plan

### Day 1-2: Clarify idea and define scope
- Finalize problem statement.
- Choose target users.
- Outline the solution.
- Decide the AI component.
- Assign tasks in the team.

### Day 3-4: Build the prototype
- Start technical development.
- Build the core user flow.
- Integrate AI and test it.
- Save progress in a repo.

### Day 5-6: Prepare materials and submit
- Record the demo video.
- Create the presentation deck.
- Write the project description.
- Submit on Devpost.

## Submission checklist

- [ ] Working prototype or demo
- [ ] Clear problem statement
- [ ] AI component explained
- [ ] Demo video recorded
- [ ] Pitch deck created
- [ ] Repository link ready
- [ ] Devpost entry completed

## Notes

- Only 10–15 teams advance to the final day.
- Focus on execution, not too many advanced features.
- Align the team name with the idea.
- If you have multiple teams, choose one.

Good luck! Stay focused on building a clear, functioning project with real civic impact.
